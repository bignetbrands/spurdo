This repo holds two things:

1. **Landing page** at `spurdosparde.fun` — the original static `index.html` (now in `public/landing.html`)
2. **Bot system** at `spurdosparde.fun/bot` — autonomous X account management for `@spurdo`

## Architecture

A Next.js 15 app deployed to Vercel. Vercel Cron triggers two scheduled jobs (one for original posts, one for replies). State lives in Upstash Redis. Content is generated by Anthropic Claude using the character bible in `/config/spurdo/`. Images are generated by OpenAI gpt-image-1 with a reference image for character consistency. The X account is posted to via the X API v2 (Basic tier).

The codebase is **project-agnostic** — set `PROJECT=spurdo` to load `/config/spurdo/`, set it to anything else to host a different character account from the same code. See [DEPLOY.md](./DEPLOY.md) for the duplicability mechanism.

## Repo layout

```
config/spurdo/             ← character bible, pillars, voice, image prompts, accounts, token
public/                    ← static landing page + reference images
src/app/                   ← Next.js routes (/bot dashboard + /api endpoints)
src/lib/                   ← config loader, KV store, auth helpers
src/types/                 ← TypeScript types
vercel.json                ← cron schedules
next.config.ts             ← / → /landing.html rewrite
```
